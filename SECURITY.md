# Security policy

Do not open a public issue for a suspected vulnerability. Send a private report
to the repository owners through GitHub's private vulnerability reporting
feature. Include affected versions, reproduction steps, impact, and any known
mitigation. Maintainers will acknowledge receipt, coordinate a fix and release,
and disclose details only after supported users can update.

Only the latest release line receives security fixes. Crash reports can contain
sensitive process data; attach the smallest redacted fixture needed to reproduce
the issue and never submit production crash artifacts.
